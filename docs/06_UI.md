# SportMentalCoach — UI (Phase 1)

UI is built with Next.js App Router, Tailwind CSS and shadcn/ui. All
domain-facing copy and taxonomy labels come from the active vertical config
(`lib/core/config`), never hardcoded inline.

---

## Surfaces

| Surface        | Route                  | Group           | Access |
|----------------|------------------------|-----------------|--------|
| Home           | `/`                    | `(dashboard)`   | public |
| Coach listing  | `/coaches`             | `(marketplace)` | public |
| Coach profile  | `/coaches/[slug]`      | `(marketplace)` | public |
| Auth           | `/sign-in`, `/sign-up` | `(login)`       | public |
| Dashboards     | `/dashboard/*`         | `(dashboard)`   | gated  |

The `(marketplace)` route group has its own minimal public header
(`app/(marketplace)/layout.tsx`): brand name + link to the coach listing +
"Accedi". It does not load the authenticated user menu.

---

## Coach marketplace (Phase 1)

### `/coaches` — listing

- Server component, `dynamic = 'force-dynamic'` (reads `searchParams`).
- Data: `getApprovedCoaches({ sport?, specialty? })` from `lib/core/listings`.
  Only `provider_profiles.status = 'approved'` rows with a slug are shown,
  joined to `profiles` for display name / avatar.
- **Filters**: a plain `<form method="get">` with two `<select>`s — sport
  (`categories` taxonomy) and specialty (`specialties` taxonomy) — sourced from
  the vertical config. Submitting reloads `/coaches?sport=…&specialty=…`. No
  client JS. An "Azzera" link clears filters when any are active.
- Each result is a `Card` showing display name, headline, sport + specialty
  chips (labels resolved via the vertical taxonomy), hourly rate, and a
  "Vedi profilo" link to the detail page.
- Empty state uses `copy['listing.empty']`.

### `/coaches/[slug]` — profile

- Server component; `getCoachBySlug(slug)` returns the approved provider with
  its `profiles` row and active `services`, or `null` → `notFound()`.
- Shows headline, description/bio, hourly rate, sport and specialty chips, and
  a list of service cards (title, duration, price).
- **Placeholder CTA**: a disabled `Button` with `copy['booking.cta']`
  ("Richiedi sessione") and a note that booking is coming soon. Booking is not
  implemented in this step.

## Public homepage & navigation

- **Homepage** (`/`, `app/(dashboard)/page.tsx`): a Kai Pai marketing page —
  hero with the brand + tagline (from the vertical config) and two CTAs
  (**Trova un coach** → `/coaches`, **Inizia ora** → `/sign-up`), a "Come
  funziona" 3-step section, an audiences section (atleti / coach / club), and a
  closing CTA band. No Stripe/pricing references, so it renders unchanged with
  billing disabled.
- **Public header** (`(dashboard)` + `(marketplace)` layouts): brand → `/`,
  a **Trova un coach** link → `/coaches`, and sign-in / sign-up (or the user
  menu). Pricing only appears when `BILLING_ENABLED`.
- **Footer**: present site-wide (see below).
- **Demo accounts** (`components/demo-accounts.tsx`): a dev-only box
  (`process.env.NODE_ENV !== 'production'`) listing seeded credentials (admin +
  demo coaches). Shown on the homepage and the sign-in page; never ships to
  production.

## Dashboard navigation

- The dashboard sidebar (`app/(dashboard)/dashboard/layout.tsx`) is role-aware:
  it fetches the current user's roles from `/api/user/roles` and renders a link
  to each role dashboard the user holds (athlete / coach / club / admin, labels
  from the vertical config), above the account settings links. Route mapping
  lives in `lib/core/auth/role-routes.ts` (a pure, non-`server-only` module so
  it is safe to import from the client).

## Branding

- The product is **Kai Pai**. The brand name comes from the vertical config
  (`copy['brand.name']`); headers and the footer read it from there. The page
  `<title>` is set in `app/layout.tsx`.

## Avatars / profile photos

- `CoachAvatar` (`components/coach-visuals.tsx`) renders `profiles.avatar_url`
  via the shadcn `Avatar`, falling back to initials (`initials()` in
  `lib/core/format.ts`). Used on coach cards, the coach profile header, and the
  dashboards.
- Athletes and coaches set their photo from their dashboard via `PhotoForm`
  (`app/(dashboard)/dashboard/photo-form.tsx`) → `updatePhotoAction` →
  `setAvatarUrl()`. Phase 1 stores an image **URL** (no file upload backend
  yet); the field validates as a URL and an empty value clears the photo.

## Certification badge

- `CertifiedBadge` shows a `BadgeCheck` icon, coloured orange when the coach is
  Kai Pai Academy certified (`provider_profiles.is_kaipai_certified`) and greyed
  out otherwise, with a `title`/`aria-label` from `copy['coach.certified.yes'|
  'no']`. Shown on each listing card and (with label) on the profile header.

## Footer

- `Footer` (`components/footer.tsx`) is a standard site footer with brand +
  tagline, contacts (email / phone / location) and social links
  (Instagram / Facebook / LinkedIn / YouTube). Rendered by the `(marketplace)`
  and `(dashboard)` layouts.

## Dashboards (Phase 1)

All dashboards are role-guarded server components under `/dashboard/*` and share
the settings chrome. Each shows the **PhotoForm** (set avatar URL).

### Athlete — `/dashboard/athlete`

- Grouped booking sections: **In attesa** (requested), **Accettate** (accepted),
  **Storico** (declined / cancelled / completed). Empty sections are hidden.
- A full empty state (no requests) shows a dashed-border card with a
  "Trova un coach" CTA to `/coaches`.

### Coach — `/dashboard/coach`

- **Status banner** (`provider_profiles.status`): coloured by state with
  guidance. When `draft` or `rejected` it shows an **"Invia per la revisione"**
  button (`submitForReviewAction` → `draft|rejected → pending`).
- **ProfileEditor** (client, `useActionState`): edits headline, bio
  (`provider_profiles.description`), sports (`categories`) and specialties as
  checkbox groups sourced from the vertical taxonomy. Saving never changes the
  status — an `approved` profile stays approved (edits go live), a non-approved
  one stays non-public until the admin approves.
- **ServicesEditor**: per-service edit/delete forms + an add form. Price is
  entered in euros and stored as cents. All ops are coach-ownership-checked in
  `lib/core/services`.
- Booking requests (pending + history) with Accetta / Rifiuta, as before.

## Conventions

- Prices are stored as integer cents; rendered with `formatPrice()`
  (`lib/core/format.ts`, `Intl.NumberFormat`, default `it-IT` / `EUR`).
- Taxonomy keys are persisted; labels are always resolved at render time via
  `findTaxonomyItem()` so a vertical swap changes all labels.
- No payment UI in Phase 1.
